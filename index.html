# Openreach Exchange Planner - GitHub Hosting Setup ## Project Structure ``` openreach-exchange-planner/ ├── index.html ├── README.md └── .gitignore ``` ## Step 1: Create index.html Create a file named `index.html` with the following content: ```html Openreach Exchange Migration Planner
``` ## Step 2: Create README.md ```markdown # Openreach Exchange Migration Planner An interactive dashboard for planning Openreach telephone exchange closures and migrations to Openreach Handover Points (OHPs). ## Features - **Geospatial Visualization**: Interactive UK map showing all exchanges and receiving sites - **Power Requirement Management**: Check if receiving sites have sufficient power capacity - **Smart Assignment System**: Click exchanges and receiving sites to create assignments - **Alternative Site Suggestions**: Automatically suggests suitable alternative sites when hovering - **Real-time Dashboard**: Track assigned vs unassigned exchanges ## Live Demo Visit: `https://yourusername.github.io/openreach-exchange-planner` ## Based on Data From This planner uses Phase 1 exchange closure data from Openreach's Exchange Exit Programme: - [ISPreview Article](https://www.ispreview.co.uk/index.php/2025/09/openreach-to-start-exiting-next-12-uk-exchanges-in-q3-2025-26.html) - [Openreach Official Programme](https://www.openreach.co.uk/cpportal/products/the-all-ip-programme/exchange-exit-programme) ## Phase 1 Exchanges (Closure by Sep 2028) 1. Staines 2. Thames Ditton 3. Baynard 4. Wraysbury 5. Nazeing 6. Langford 7. Allestree Park 8. Beacon 9. Childwall 10. Lundin Links 11. Carrickfergus 12. Glengormley ## How to Use 1. **Select an Exchange**: Click on any exchange in the left panel or on the map 2. **View Power Requirements**: See the power requirement for the selected exchange 3. **Hover Over Sites**: Hover over receiving sites (OHPs) to see if they can handle the exchange 4. **Assign Exchange**: Click on a suitable receiving site to create the assignment 5. **View Alternatives**: Check the bottom panel for all available receiving sites sorted by distance ### Color Coding - 🟠 **Orange Circle**: Unassigned exchange - 🟢 **Green Circle**: Assigned exchange - 🔵 **Blue Square**: Receiving site (OHP) - 🔴 **Red Square**: Receiving site cannot handle selected exchange (insufficient power) - 🟡 **Yellow Square**: Receiving site at capacity ## Local Development Simply open `index.html` in a web browser. No build process required! ## Technologies Used - React 18 - Tailwind CSS - Vanilla JavaScript (ES6+) - SVG for map visualization ## License MIT License - feel free to use and modify ## Contributing Pull requests are welcome! For major changes, please open an issue first. ``` ## Step 3: Create .gitignore ``` # OS Files .DS_Store Thumbs.db # Editor files .vscode/ .idea/ *.swp *.swo *~ # Logs *.log npm-debug.log* ``` ## Step 4: Deploy to GitHub Pages 1. **Create a new repository on GitHub** - Go to https://github.com/new - Name it `openreach-exchange-planner` - Make it public - Don't initialize with README (we'll add our own) 2. **Push your code** ```bash # Navigate to your project folder cd openreach-exchange-planner # Initialize git git init # Add all files git add . # Commit git commit -m "Initial commit: Openreach Exchange Planner" # Add remote (replace YOUR_USERNAME) git remote add origin https://github.com/YOUR_USERNAME/openreach-exchange-planner.git # Push to GitHub git branch -M main git push -u origin main ``` 3. **Enable GitHub Pages** - Go to your repository on GitHub - Click **Settings** tab - Scroll to **Pages** section (left sidebar) - Under "Source", select **main** branch - Select **/ (root)** folder - Click **Save** - Wait 1-2 minutes for deployment 4. **Access your site** - Your site will be live at: `https://YOUR_USERNAME.github.io/openreach-exchange-planner/` ## Alternative: Quick Deploy Commands ```bash # Clone or create your directory mkdir openreach-exchange-planner cd openreach-exchange-planner # Create the files (copy content from above) # - index.html # - README.md # - .gitignore # Initialize and push git init git add . git commit -m "Initial commit" git remote add origin https://github.com/YOUR_USERNAME/openreach-exchange-planner.git git push -u origin main ``` ## Customization Ideas 1. **Add more exchanges**: Edit the `exchangeData.phase1` array in `index.html` 2. **Add more receiving sites**: Edit the `receivingSites` array 3. **Change color scheme**: Modify Tailwind classes 4. **Add export functionality**: Add buttons to export assignments as JSON/CSV 5. **Add authentication**: Integrate GitHub OAuth for saving user assignments ## Troubleshooting **Page not loading?** - Check that GitHub Pages is enabled in repository settings - Make sure `index.html` is in the root directory - Wait a few minutes for deployment to complete **Styles not working?** - The CDN links for React and Tailwind should work automatically - Check browser console for any errors **Need help?** - Open an issue on GitHub - Check the browser console for error messages
